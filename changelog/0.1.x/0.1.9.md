# 0.1.9 — 2026-05-28

## Changed

- **`invalid_radius` error code** reclassified from `InvalidParams` (-32602) to `ValidationError` (-32007) in `earthquake_count` and `earthquake_search` — the condition is domain validation (coordinate/radius fields provided incompletely), not a malformed JSON-RPC parameter.
- **`landing.requireAuth: false`** set explicitly in `createApp()` — required by `@cyanheads/mcp-ts-core` 0.9.13, which now gates the landing-page tool/resource inventory behind auth when `MCP_AUTH_MODE` is `jwt` or `oauth`. Public-catalog deployments must opt out explicitly; this server does.
- **`@cyanheads/mcp-ts-core`** ^0.9.9 → ^0.9.13:
  - **HTTP 413 body cap** (`MCP_HTTP_MAX_BODY_BYTES`) — oversized inbound requests rejected before body parse; default 1 MiB.
  - **HTTP session-init gate** — stateful HTTP mode now rejects non-`initialize` requests arriving without `Mcp-Session-Id` with HTTP 400.
  - **Client-error log level** — 401/403/400/404 responses now log at `warning` instead of running the full error pipeline with stack traces.
  - **`GET /mcp` keywords** — `package.json` `keywords` array now surfaced on the status endpoint alongside `name`, `version`, and `description`.
- **`@biomejs/biome`** ^2.4.15 → ^2.4.16
