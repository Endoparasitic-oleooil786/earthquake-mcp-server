# 0.1.8 — 2026-05-24

## Changed

- **`makeReqCtx` helper** extracted in `usgs-service.ts` and `emsc-service.ts` — four inline `requestContextService.createRequestContext()` blocks per file replaced with one shared helper.
- **`isTier` / `isWindow` type predicates** in `earthquake-feed.resource.ts` replace cast-and-check patterns, removing manual `as (typeof VALID_TIERS)[number]` casts from the handler.
- **`EarthquakeEventOutput`** type in `schemas.ts` replaced manual type declaration with `z.infer<typeof EarthquakeEventSchema>` — single source of truth, no sync risk.
- **`query_too_broad` error code** corrected from `InvalidParams` (-32602) to `ValidationError` (-32007) — the condition is domain validation (result set too large), not a malformed parameter.
- **`git-wrapup` skill** added to `skills/` and registered in `CLAUDE.md` skills table.
- **`release-and-publish` skill** updated to v2.6 — push branch before tags, MCP Registry URL corrected to `/v0.1/` path, docker auth recovery, idempotent-success handling.
- **CLAUDE.md** updated: framework reference `^0.9.7` → `^0.9.9`, "Close the loop on issues" core rule added, context table entries for `ctx.fail` / `ctx.notifyPromptListChanged` / `ctx.notifyToolListChanged` added.
- **`@cyanheads/mcp-ts-core`** ^0.9.7 → ^0.9.9 — fuzz-engine pre-parse fix (false-positive crashes from `.default()` / `.min()`), `ZodArray` minLength now passed to `fc.array()`, `ctx.elicit` example corrected (`result.content.format`), `git-wrapup` skill added upstream.
